How ARIA Works: Chat Flow & Capabilities

Understand the step-by-step pipeline when you ask ARIA a question about your data.

ARIA is not just an LLM that blindly writes SQL. It is a highly governed, multi-stage pipeline designed to protect your database while delivering accurate insights and visualizations in seconds.

💡 Did you know?

ARIA uses Qdrant and Mem0 to remember your previous questions, your team's business rules, and your preferred chart types. The more you use it, the smarter it gets.

The Query Pipeline (Server-Sent Events)

When you type a question like "Show me monthly revenue by region", ARIA streams its progress back to your screen instantly. Here is what happens under the hood:

  1. THINKING: ARIA analyzes your intent. Is it a new question, or are you correcting a previous chart?
  2. GENERATING_SQL: ARIA searches the Vault (Markdown documentation of your database) and Team Memory. If the question is simple, it uses a fast Rule-Based generator. If complex, it falls back to the LLM.
  3. SQL_READY: The SQL is generated. (Administrators can view the raw SQL code via the UI).
  4. EXECUTING:
    • Guard: ARIA checks if the query is a safe SELECT statement.
    • Dry-Run: ARIA estimates the row count (EXPLAIN). If it exceeds your Tenant's Max Row Limit, it blocks the query to save your database from crashing.
  5. RENDERING_CHART: The data returns. ARIA automatically chooses the best chart (Bar, Line, Pie) or uses your preferred type. It also exports a CSV to MinIO.
  6. INSIGHT_GENERATION: The LLM reads the result table and generates an Executive Summary and 3 clickable Follow-up Suggestions.
⚠️ Large Data Exports

If your query returns too many rows for the browser (e.g., >5,000 rows), ARIA will not crash your tab. Instead, it offloads the process to a Prefect Background Job and provides you with a downloadable 3-day MinIO link.

Vault Auto-Sync

ARIA reads from your live database. If your data engineering team adds a new column to the PostgreSQL or Oracle database, ARIA won't fall behind.

  • Navigate to Admin > Data Dictionary.
  • Click the Re-Sync button.
  • ARIA will safely diff the live database schema against the Markdown vault, adding new columns without deleting your human-written business descriptions.
