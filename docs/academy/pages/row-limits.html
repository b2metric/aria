Row Limits & Safeguards

Understand how ARIA prevents massive queries from destroying your databases.

A Conversational BI tool has the power to unleash an LLM on your production databases. Left unchecked, an AI might generate a query that attempts to select 10 million rows, crashing your server.

ARIA has built-in EXPLAIN dry-runs and safeguards to prevent this.

1. The SELECT-Only Guard

All generated SQL passes through an strict regex blocker. If the LLM produces statements like DROP, DELETE, or UPDATE, the system will throw a Security Exception before it even reaches the database execution engine.

2. EXPLAIN Row Estimation

Before running a query, ARIA prefixes it with EXPLAIN (FORMAT JSON) on PostgreSQL or EXPLAIN PLAN on Oracle.

If the estimated_rows returned by the database planner is drastically higher than your Tenant's Max Row Limit (e.g. 1,000,000 estimated vs. 1,000 limit), ARIA immediately blocks the query.

💡 Tenant Configuration

You can configure the Max Row Limit under the Admin > Tenant Config page. The default is usually 1,000 rows.
