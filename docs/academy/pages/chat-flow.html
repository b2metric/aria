Chat Flow & Pipelines

Deep dive into ARIA's Natural Language to SQL generation engine.

When interacting with ARIA, the system uses a heuristic rule engine supported by an LLM fallback. This hybrid approach guarantees 100% syntactic accuracy for standard queries while leveraging AI only for complex multi-table joins.

1. The Heuristic Engine (Rule-based)

ARIA scans your database Vault schemas (Data Dictionary) to find column names matching the keywords in your question.

If User asks: "Show me total revenue"
Rule Engine: Looks for columns containing 'revenue', 'amount', or 'total'
Result: Executes SQL natively (0 Tokens Used)

2. LLM Fallback (Generative AI)

If the rule engine fails to confidently match your question to a specific table (Score < 15), ARIA sends your schema context and the question to an LLM (e.g., Claude or Gemini via LiteLLM proxy).

💡 Self-Correction

If the LLM generates a SQL query with a syntax error, ARIA does not show an error to the user immediately. It catches the error in the backend, sends the error message back to the LLM, and asks it to fix the SQL automatically.

Understanding the SSE Stream

ARIA utilizes Server-Sent Events (SSE) to update the chat interface. You will see these statuses:

  • Thinking: Parsing intent
  • Generating SQL: Running rule engine or calling LLM
  • Executing: Running the query against your Oracle/PostgreSQL DB
  • Rendering Chart: Building UI configurations
  • Generating Insight: Calling the LLM to write a summary and suggestions
